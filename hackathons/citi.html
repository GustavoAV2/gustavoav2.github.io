Desafio proposto

A proposta do desafio era a criação de um produto que permita:

  • Receber arquivo de REMESSA1 da empresa que solicita geração de QR Code para cobrança, formato CNAB 7502 (Layout Febraban )
  • Transformar cada solicitação de geração de QR Code, recebida no arquivo, em mensagens aceitas por esse formato : (https://github.com/bacen/pix-api)
  • Após o processamento das solicitações de geração de QR Code, transformar cada resposta ( ) no formato correspondente ao arquivo de RETORNO , formato CNAB 750;

Requisitos Funcionais e Negócio

Para processar o arquivo REMESSA CNAB 750, precisará ser previsto duas formas:

Normal:
  • Arquivo deverá ser processado automaticamente assim que estiver disponível em um diretório;
Contingência:
  • Ter uma interface que permita o operador fazer o upload do arquivo remessa e permita fazer download dos arquivos de retorno gerados pela aplicação;
Volumetria para processamento
  • Criação de uma solução que permita processar 100 mil ou mais solicitações de QRCode por hora.

Solução apresentada

Processador de arquivos CNAB
API de cobrança

Esta é a API principal para o processamento de Remessa e a criação das cobranças e QRCode. A API recebe requisições tanto da Interface FrontEnd pelo usuário, quanto do worker que está monitorando o diretório de arquivos de remessa. A API tem a responsabilidade de interpretar os arquivos, gerar o QRCode de cobrança e criar as respostas correspondentes no formato do arquivo de retorno CNAB 750. Após o processamento, o arquivo de retorno é salvo em um diretório de saída.

Monitoramento de diretório de arquivos de remessa

Este projeto consiste em um script Python que utiliza a biblioteca watchdog para monitorar um diretório específico em busca de novos arquivos de remessa. Quando um novo arquivo é detectado, o arquivo é coletado e enviado para a API De Cobrança onde esse arquivo será processado. Este processador é responsavel por monitorar e encaminhar os arquivos a API, que por sua vez tem a responsabilidade de interpretar os arquivos, gerar o QRCode de cobrança e criar as respostas correspondentes no formato do arquivo de retorno CNAB 750. Após o processamento, o arquivo de retorno é salvo em um diretório de saída.
